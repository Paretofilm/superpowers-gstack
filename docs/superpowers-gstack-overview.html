Architecture in 4 layers

Each layer is independently versioned and replaceable. superpowers-gstack only touches its own layer (the plugin) and the project CLAUDE.md generated for end-users — it never modifies upstream skills.

L4 → L3 Routes

Project CLAUDE.md instructs Claude to invoke specific superpowers-gstack skills based on user phrasing.

L3 → L1+L2 Composes

Our skills call upstream skills via the Skill tool. We don't fork; we orchestrate.

L0 ← All Executes

Claude Code runs all bash, manages skill loading, dispatches subagents.

The 13 skills

Each skill is a markdown instruction set Claude reads and follows. No executable runtime — composition over reimplementation.

Routing & Setup

/setup-routing

Asks about project type, evaluates relevant skills from both Superpowers and GStack, and emits a tailored CLAUDE.md routing block.

/adapt

Adapts an existing project (one without a CLAUDE.md) by analyzing it, identifying gaps, and adding routing without losing existing content.

Plan Quality

/pitfall-verification

Targeted check that typical pitfalls (security, idempotency, contracts, edge cases, LLM output) do not apply to a finished PRD, spec, plan, or code artifact.

/quality-review

Perceived-quality gate — hunts pitfalls that make a product feel cheap or broken even when it technically works.

Native Apple

/macos-native-review

HIG-citation-grounded conformance gate for macOS apps. 12 categories: vocabulary, controls, shortcuts, menu bar, lifecycle, etc.

/ios-native-review

Same shape, iOS-specific surfaces: touch targets, navigation paradigm, modal presentation, Dynamic Island, haptics.

/swiftui-design-consultation

Produces DESIGN.md + Swift Package starter. Inlines the platform question (iOS/macOS/both) on first run.

/macos-e2e-scaffold

E2E test scaffold for macOS apps.

Workflow Acceleration

/autoimplement v2.14.2

Auto-advance through a multi-phase plan. Dispatches subagent per phase, chains review + pitfall + codex at boundaries. Active pre-flight reviews the plan body itself before Phase 1 unless a marker commit exists.

/office-hours-track-aware

Wraps upstream /office-hours with track inference (web vs native), inline platform question, approve-before-render gate.

Context & Artifacts

/context-handoff

Structured handoff file (YAML frontmatter + prose) to docs/superpowers/handoff.md before /clear or /compact. Survives across machines.

/htmlify

Beautiful, pedagogical HTML companions for MD artifacts. Auto-opens via --open. Per-directory dashboards.

A typical workflow

From idea to shipped feature, with review gates at every boundary. Cross-model adversarial (Claude + Codex) at the strongest gates.

Cross-model adversarial review

Claude reviews catch one class of bugs. Codex catches another. Combined, they find structural blind-spots single-model review systematically misses.

Empirical evidence from this plugin's own development (May 2026)

4 of the last 6 PRs landed bugs that codex caught and Claude's review/pitfall missed. Each bug was structurally invisible to single-model review.

PRBugCaught bySingle-model blind-spot
#24 zsh portability — bare status= is read-only var user live dogfood Bash snippets in SKILL.md execute via Claude Code's Bash tool with user $SHELL (zsh on macOS), not bash
#25 Fixture cross-run idempotency — grep -c == 1 breaks on re-runs codex Cross-run/temporal correctness is invisible to single-diff review
#26 4 P1/P2 in pre-flight design (empty commit marker, latest vs historical, codex-unavailable lie, advisory gap) codex × 3 rounds Cross-section contract violations in a long SKILL.md spec
#28 Marker regex precision (4 attempts to converge) codex × 9 rounds Subtle regex bypass cases + doc-consistency drift across multiple files

Automated upstream pipeline

A weekly GitHub Action checks upstream Superpowers, GStack, and Claude Code for changes. If anything moved, Claude API auto-generates a PR with the manual updated. Self-repair on failures.

Annual cost

~$2.60–$10 (52 weekly Sonnet calls × ~$0.05–$0.20 each). Charged to your ANTHROPIC_API_KEY secret.

Max plan not touched

Runtime skill usage uses your session credentials (Max plan). Only the weekly cron and self-repair use the API key.

Self-healing

If check-updates.yml fails, self-repair.yml diagnoses via Claude API and opens a fix PR automatically.

Plugin lifecycle

Three commands you'll run. Restart alone is not enough — the cache is version-keyed.

1 · Install

One time:

# in Claude Code
/plugin marketplace add \
  Paretofilm/claude-marketplace

/plugin install \
  superpowers-gstack@paretofilm-plugins

2 · Update

When new version ships:

# pulls latest cache
/plugin update \
  superpowers-gstack@paretofilm-plugins

# syncs live router
/reload-plugins

3 · Routing

Per new project:

# bootstrap CLAUDE.md
/superpowers-gstack:setup-routing

# or for existing project
/superpowers-gstack:adapt

Every skill carries its own audit trail

High-trust skills (autoimplement, pitfall-verification) ship with an audit-trail section documenting which reviews ran on the skill itself before it was trusted to run on user code.

Why this matters

The audit trail makes trust auditable. When a skill is "high-trust" (runs autonomously without human checkpoints), users can verify this specific skill went through the review chain — by reading the audit trail in the shipped SKILL.md itself.
